# Contributing to Cursed Tomorrow

Ahmad Hassan (B-Ted) appreciates the interest in contributing to *Cursed Tomorrow*. This project is a space for intentional game engineering, and new people are welcomed to help improve it.

## How to Contribute

### Reporting Issues
- Check the issue tracker for existing reports.
- If the issue is new, open a clear report with reproduction steps.

### Improvements
- Enhancement suggestions are always considered. Please describe the motivation and impact of the proposed change.

### Pull Requests
1. Fork the repository.
2. Create a feature branch.
3. Ensure the code follows the `.editorconfig` standards.
4. Submit the PR with a clear description of the changes.

## Engineering Standards
- Use 4 spaces for indentation.
- Follow the established namespace structure (`CursedTomorrow::*`).
- Prioritize readability and performance.

The goal is to maintain a professional and technically sound codebase.
